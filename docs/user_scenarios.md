# 마이링크 (MyLink) - 사용자 시나리오 (User Scenarios)

> **작성 형식 규칙**: "사용자는 __하기 위해 __한다"

---

## 1. 방문자 (Visitor) 시나리오

### 1.1 마이링크 프로필 접근 및 상호작용
- **방문자는** 크리에이터(소유자)의 흩어진 SNS 채널과 콘텐츠를 한곳에서 파악하기 위해, 다른 소셜 플랫폼에 등록된 마이링크 주소(`mylink.com/닉네임`)를 클릭하여 랜딩 페이지에 **접속한다**.
- **방문자는** 해당 페이지가 본인이 찾던 크리에이터의 공식 페이지인지 빠르게 파악하기 위해, 상단에 노출된 구글 연동 프로필 이미지(`photoUrl`), 닉네임(`displayName`), 짧은 소개글(`Bio`) 정보를 **열람한다**.
- **방문자는** 자신이 원하는 특정 콘텐츠(예: 포트폴리오 사이트, 영상, 예약창 등)로 정확히 이동하기 위해, 제공된 여러 링크 목록 중에서 직관적인 텍스트 제목과 구글 파비콘이 렌더링된 버튼을 **클릭한다**.
- **방문자는** 연결된 목적지 사이트에서 필요한 작업(결제, 시청, 문의 등)을 수행하기 위해, 새 창으로 열린 웹사이트를 적극적으로 **탐색한다**.

---

## 2. 소유자 (Owner) 시나리오

### 2.1 회원가입 및 로그인
- **소유자는** 복잡한 정보 입력 과정 없이 안전하게 서비스에 합류하기 위해, 메인 페이지 상에서 '구글 소셜 로그인' 버튼을 **클릭한다**.
- **소유자는** 자신만의 고유한 브랜딩 및 개인용 접속 주소를 확보하기 위해, 최초 가입 화면에서 고유한 닉네임(displayName)을 **입력한다**. (이 닉네임은 `mylink.com/닉네임` 형태로 URL에 매핑됨)

### 2.2 프로필 관리 (인라인 편집)
- **소유자는** 자신의 공식 얼굴(아바타)을 방문자에게 보여주기 위해, 프로필 이미지를 직접 업로드하는 대신 설정된 구글 계정의 프로필 사진이 상단 로고로 렌더링 되었음을 **확인한다**.
- **소유자는** 자신을 설명하는 소개글(Bio)을 사용자에게 좀 더 매력적으로 전달하기 위해, 대시보드 상단 소개글 텍스트 부분을 직접 클릭하여 활성화된 **인라인 편집(Inline Edit)** 모드에서 내용을 수정하고 즉시 **저장한다**.
- **소유자는** 닉네임과 이에 연동된 접속 주소 URL을 함께 변경하기 위해, 닉네임 텍스트를 클릭하여 인라인 상태로 수정하고 엔터(Enter)를 입력하여 **변경을 완료한다**.

### 2.3 링크 추가, 수정 및 삭제 (Link Management)
- **소유자는** 자신의 새로운 콘텐츠(유튜브 영상, 제품 상세 페이지 등)를 방문자에게 홍보하기 위해, 대시보드 화면 하단의 '+ 링크 추가' 버튼을 빈 슬롯을 생성하기 위해 **클릭한다**.
- **소유자는** 추가한 링크의 목적과 연결할 목적지를 방문자에게 제공하기 위해, 생성된 빈 카드의 텍스트 영역(제목, URL)을 차례대로 클릭하고 내용을 인라인으로 **입력 및 저장한다**.
- **소유자는** 방문자가 시각적으로 링크의 출처를 쉽게 파악할 수 있도록 하기 위해, 구글 API 연동을 통해 시스템이 방금 입력한 URL의 파비콘(Favicon)을 링크 아이템의 로고로 자동 렌더링함을 **확인한다**.
- **소유자는** 예전에 등록해둔 링크의 목적지 주소가 변경되었거나 목적지 타이틀을 수정하기 위해, 기존 링크 카드의 텍스트 부분을 즉시 클릭하고 내용을 인라인으로 **수정(Edit)한다**.
- **소유자는** 더 이상 홍보가 필요 없거나 이벤트가 끝난 링크를 영구적으로 노출에서 제거하기 위해, 해당 링크 카드의 우측에 있는 '삭제(휴지통)' 아이콘을 **클릭한다**.
- **소유자는** 실수로 클릭하여 소중한 링크 정보를 날려버리는 것을 방지하기 위해, 삭제 알림성 팝업/모달 창에서 최종 확인 버튼을 **승인한다**.
